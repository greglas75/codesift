## Critical Lesson: `references` Is a Budget Bomb

R18 attempt 1 failed because `references("createRisk")` consumed 19,637 tokens — the symbol appears in test helpers, production code, and fixture functions across 40+ files. A single `references` query for a popular symbol exhausts the entire 20K budget.

**Rule:** Never use `references` in a shared-budget call unless the symbol is rare (unique to 1-3 files).

## Remaining Gap Analysis

| Source | Tokens |
|--------|--------|
| Extra tool calls (11 vs Auggie's 5) | ~3,000 |
| Model planning verbosity | ~2,000 |
| **Total gap** | **~5,031** |

The overhead is structural — ToolSearch + list_repos + 2 codebase_retrieval calls = 11 tool uses vs Auggie's 5. Each tool call and response adds ~500-800 tokens to the conversation history that accumulates across all subsequent turns.

## Path to Beat Auggie (#1)

To close the 5,031 token gap:
1. **Eliminate ToolSearch overhead**: if agent can use codebase_retrieval without ToolSearch first → saves 1 tool call
2. **Combine list_repos + first call somehow**: saves 1 tool call
3. **Shorter model answer style**: the agent writes very detailed answers (good for quality, bad for tokens)
4. **Pre-baked repo name**: if agent knows `"local/promptvault"` without list_repos call → saves 1 tool call

Realistic theoretical minimum with current architecture: ~65-66K tokens (saves ToolSearch + list_repos = 2 tool call overhead ≈ 3K tokens).
