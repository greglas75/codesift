## Observations

### 1. Auggie new all-time record: 64,148 tokens
Previous record was 70,508 (R8 Hybrid). Auggie batched all 10 tasks into 4 codebase-retrieval calls with 0 follow-ups. Semantic search continues to improve with batching.

### 2. CodeSift BM25: 84K — above target (79,791)
First CodeSift run on live promptvault data. 12 self-reported mcp__codesift__ calls (vs Munch's 5 in this run). Several issues to fix:
- Cold WASM parse overhead on first queries
- BM25 token output may be more verbose than jcodemunch's compact JSON
- Possible: codebase_retrieval returning more results than needed (higher top_k)

**Gap to close:** 84K → target 79K = ~6% improvement needed.

### 3. Munch R10 outlier: 125K (avg historical: 99K)
Agent made only 5 self-reported calls but 29 system calls — likely poor batching strategy or large codebase_retrieval responses. Not representative of Munch's typical performance.

### 4. Standard strong: 74,983 tokens
Consistent with R7-R9 range (80-81K). Slightly better this run, possibly due to CLAUDE.md architecture overview.

## CodeSift vs Munch (BM25 parity check)

| Metric | CodeSift BM25 | Munch R10 | Munch avg (R1-R9) |
|--------|--------------|-----------|-------------------|
| Tokens | 84,051 | 125,849 | 99,231 |
| Duration | 183s | 230s | 197s |
| Self calls | 12 | 5 | ~18 (R6b) |

CodeSift faster on self-calls (12 vs historical 18 for Munch) but slightly higher tokens. The gap is likely due to:
1. Response verbosity differences (CodeSift TypeScript vs Munch Python serialization)
2. WASM cold-start overhead added to first batch query duration
3. Munch has more mature BM25 tuning (after 9 rounds of optimization)
