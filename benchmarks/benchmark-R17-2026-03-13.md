## Critical Finding: Query Type Selection

**Semantic** = conceptual, architectural, pattern-matching (T4, T5, T6, T10)
**Text** = exact strings, function names, error codes, specific APIs (T1, T7, T9)

| Task type | Best query type | Token cost | Coverage |
|-----------|----------------|-----------|---------|
| Find exact function by name | `text "async function X" file_pattern:"*.service.ts"` | ~363 | 100% |
| Find exact error code + all usages | `text "ERROR_CODE"` | ~1,479 | 100% |
| Find all usages of specific API | `text "prisma.$transaction" file_pattern:"*.service.ts"` | ~1,726 | 100% |
| Understand middleware/auth flow | `semantic "withAuth middleware session"` | ~2,000 | high |
| Component props + hooks | `semantic "RiskPanel props hooks"` | ~2,000 | high |
| Architecture trace | `semantic "pipeline segmentation AI"` | ~2,000 | high |

**Rule:** If you know the exact string → use `text`. If you need conceptual understanding → use `semantic`.

## Context Accumulation Analysis

With 3 calls vs 4 calls:
- 3 calls: result data accumulates as 6K → 12K → 18K across calls
- 4 calls: result data accumulates as 6K → 12K → 18K → 24K across calls
- Saving: ~6K fewer tokens in accumulated context → ~6K total token reduction

Main remaining overhead vs Auggie (64,148): ~7,270 tokens of agent planning/thinking verbosity.

## Path to Beat Auggie on Tokens

Gap: 7,270 tokens. Possible levers:
1. **2 calls instead of 3** — save ~6K tokens in accumulation (but risky: need to fit more queries per call without truncation)
2. **Shorter model answers** — agent writes terse answers → less output tokens (hard to control)
3. **T2 improvement** — currently uses semantic (partial); use `references` sub-query type for exact import list
4. **Reduce semantic top_k=3** for simple lookups — saves ~500 tokens/query × 8 queries = ~4K tokens

Realistic target for R18: 2 calls, mixed text+semantic, top_k=3 for simple queries → **~65K tokens**
