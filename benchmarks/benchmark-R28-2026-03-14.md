# Benchmark R28: Read/Grep Fallback + Form Answers (CodeSift Unavailable)

**Date:** 2026-03-14
**Project:** promptvault (Next.js 14+, TypeScript, Prisma)
**Model:** Claude Sonnet 4.6
**Note:** INVALID as CodeSift benchmark — CodeSift unavailable, agent fell back to Read/Grep/Glob

---

## Context

Launched as parallel CodeSift benchmark but CodeSift MCP was not available in the subagent session. Agent used native Read/Grep/Glob tools with 26 tool calls. Form-style answers were requested.

---

## Results

| Metric | R28 | R27 | R10 Auggie | R10 Standard |
|--------|-----|-----|------------|--------------|
| **Total tokens** | **~63,948** | ~51,081 | 64,148 | 74,983 |
| **Tool calls** | 26 | 5-6 | 5 | ~30 |
| **Quality** | ~9.5/10 | ~8.5/10 | 9/10 | 10/10 |

---

## Key Finding: Read/Grep + Form Answers = Beats Auggie R10

R28 (63,948, native tools) vs Auggie R10 (64,148): **200 tokens fewer** despite 26 tool calls.

With form answers, even native Read/Grep beats Auggie! This confirms:
1. Form answers save ~18K vs prose — works with ALL tool types
2. Standard tools (R10 Standard) used prose → 74,983 tokens
3. Same tools + form answers → ~63,948 tokens (saves ~11K)

However, R28 still uses far more tool calls (26 vs 5) and has ~13K overhead vs CodeSift R27.

---

## Quality Notes

Native Read/Grep produces excellent quality because:
- Exact file reads, no missed chunks
- All T9 locations found precisely (actual search, not embedding approximation)
- All T7 throw sites found
- Score ~9.5/10 — better than all embedding-based approaches except R18

**The quality-token frontier:**
- Best tokens: CodeSift R27 (51,081) — 8.5/10
- Best quality: Standard R10 (74,983) — 10/10
- Native + form answers (R28): 63,948 — 9.5/10 (middle ground)

---

## Why CodeSift + Form > Read/Grep + Form

| Factor | CodeSift R27 | Read/Grep R28 |
|--------|-------------|----------------|
| Result tokens | ~19K (dense) | ~25K (verbose) |
| Tool call overhead | ~300 tokens | ~3,000 tokens |
| Answer tokens | ~3K (form) | ~3K (form) |
| **Total** | **~51K** | **~64K** |

CodeSift semantic compression beats Read/Grep verbosity even with form answers.

---

## Valid Insight: Form Answer Savings Are Tool-Agnostic

| Method | Prose | Form | Savings |
|--------|-------|------|---------|
| CodeSift | 69,179 (R18) | ~51,081 (R27) | ~18K |
| Read/Grep | ~74,983 (R10) | ~63,948 (R28) | ~11K |
| Auggie | 64,148 (R10) | TBD (R31) | expected ~15K |

Form savings vary by method because prose verbosity differs per tool's natural response pattern.
