## Critical Finding: Form Answers Don't Save ~18K for Auggie

| Method | Format | Tokens | vs R10 |
|--------|--------|--------|--------|
| Auggie R10 | prose | 64,148 | baseline |
| **Auggie R31** | **form** | **65,210** | **+1,062 (worse!)** |
| CodeSift R18 | prose | 69,179 | +5,031 |
| **CodeSift R27** | **form** | **~51,081** | **-13K vs R10** |

**Why form saves 18K for CodeSift but 0K for Auggie:**
- CodeSift returns raw code chunks → agent writes prose explanations → form format cuts 18K of explanation
- Auggie returns semantic summaries with context → agent already writes concise answers → form format saves almost nothing

The 18K form-answer saving is specific to tools that return verbose/unformatted raw code.

---

## T5 Overload: Auggie Too Broad

R31's T5 (Zod schemas) returned ALL schemas from the entire API codebase, not just risk-related ones. This is Auggie's "too helpful" behavior for broad semantic queries — returns everything remotely relevant, inflating result tokens. CodeSift's explicit top_k limits prevent this.
