## Observations

### 1. Hybrid WORSE than BM25 in tokens: 90,946 vs 84,051 (+8%)

Unexpected result — adding semantic search increased tokens. Fewer tool calls (9 vs 12) but each call returned more verbose results.

**Root causes:**
- Semantic sub-queries (`type: "semantic"`) returned full source bodies with high token cost
- Agent made broader queries when semantic was available (more confident → less targeted)
- `text-embedding-3-small` 1536-dim vectors may match too broadly → more results returned
- No verbosity tuning yet for hybrid mode (BM25 already had `trim source to 200 chars` TODO)

### 2. Duration improvement: 164s vs 183s (-10%)

Fewer tool calls (9 vs 12) saved time despite semantic overhead. The WASM cold-start from R10 is now warm.

### 3. Still far from Auggie target: 90,946 vs 64,148 (42% gap)

Phase 3 target (<64,148) not yet met. Major work needed.

## Analysis: Why Semantic Made Things Worse

| Factor | Impact |
|--------|--------|
| Semantic returns full source bodies | +5K tokens |
| Agent makes broader initial queries | +3K tokens |
| Cross-query dedup less effective with semantic hits | +2K tokens |
| No source truncation on semantic results | ~same as BM25 issue |
