## Critical Observation: High Variance Across Auggie Runs

R34 and R35 use near-identical 2-call structure. Token counts are within 200 tokens. But different tasks fail:

| Task | R34 | R35 | Reason |
|------|-----|-----|--------|
| T7_throws | 7/10 | 0/10 | Same query, different return |
| T10 pipeline | 3/10 | 9/10 | Moved to Call A |

Auggie's semantic retrieval is probabilistic — the same query can return different code chunks. CodeSift's deterministic text queries (BM25) avoid this variance.

**Quality at 51K is NOT STABLE with Auggie** — different runs yield different task failures.

---

## Why CodeSift Has Quality Advantage at Same Token Count

1. **Structured multi-query**: 4-6 sub-queries per call, each semantically focused on 1 task
2. **Text/BM25 for exact strings**: Deterministic coverage for T7, T9
3. **top_k control**: Prevents result bloat from over-relevant returns
4. **Zero variance**: Same query always returns same top-k chunks

Auggie at 2 calls with ultra-terse ≈ CodeSift at 2 calls with form in **tokens**, but CodeSift is **more reliable**.
