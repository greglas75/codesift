## Findings

### R-1 [MUST-FIX] Hono `parseFile` cache hit skips nested `app.route` expansion

- **File:** `src/parser/extractors/hono.ts` (as of `61d7d28`; pattern still present on current `HEAD` around `parseFile` cache branch).
- **Confidence:** 82/100
- **Evidence:** On cache hit, the implementation replays `cached.routes` and **returns immediately**, so `walkRouteMounts` never runs again. The first successful parse runs `walkRouteMounts` *after* `parsedCache.set`, which recursively expands child routers. A **second** mount of the same child file (different prefix) therefore gets **flat routes from `localRoutes` only**, not routes produced by nested mounts inside that child.
- **Fix:** Do not short-circuit the full mount walk on cache hit — e.g. always run `walkRouteMounts` with `(tree, importMap)` available, or key cache by `(file, prefix)` and avoid caching across mount subtrees, or replay stored mount plan + re-invoke child parses with new prefix.

### R-2 [RECOMMENDED] `HonoMount.parent_var` uses `Object.keys(appVars)[0]`

- **File:** `src/parser/extractors/hono.ts` — `walkRouteMounts` (`61d7d28`).
- **Confidence:** 71/100
- **Evidence:** First key of `appVars` is not guaranteed to be the variable that owns `app.route()`; misleading for multi-app files.
- **Fix:** Derive `parent_var` from the `ownerVar` of each `app.route()` callsite (walker should pass owner through).

### R-3 [RECOMMENDED] Memoization cache populated before mounts complete

- **File:** `src/parser/extractors/hono.ts`
- **Confidence:** 68/100
- **Evidence:** `parsedCache.set(...)` runs with `localMounts` still empty; `cached.mounts` never updated after `walkRouteMounts`. Any future consumer expecting mounts in cache reads stale data.

### R-4 [RECOMMENDED] Python partial extraction flags first symbol via loose cast

- **File:** `src/parser/extractors/python.ts`
- **Confidence:** 62/100
- **Evidence:** `partial_extraction` attached to `symbols[0]` with `(first as Record<string, unknown>).meta` — works but is type-unsafe and arbitrary for consumers.
- **Fix:** Model-level `extraction_status` / `meta` on return value instead of mutating an arbitrary symbol.

### R-5 [NIT] Cycle detection in `parseFile` returns silently

- **File:** `src/parser/extractors/hono.ts`
- **Confidence:** 58/100
- **Evidence:** `if (inFlight.has(file)) return;` does not increment `skip_reasons` — harder to debug incomplete graphs.
