# Formatter domain split review

## Scope

The 1180-line formatter monolith was split into core, graph, analysis, and Next.js modules. `src/formatters.ts` remains a compatibility facade, and tool-facing formatter selection is centralized in `src/formatter-dispatch.ts`.

## Evidence

- Independent CQ auditor: PASS, 9/11 applicable gates (81.8%), no critical-gate failure.
- Facade contract: 37/37 runtime exports preserved; normalized moved formatter bodies had 0 differences.
- Dispatch map: all 38 formatter-using tool names in core, analysis, Next.js, and meta registrations are covered and tested.
- Native scoped import check: `FORMATTER_CYCLES_NONE`.
- Verification: `npm run lint` PASS; `npm run build` PASS; focused formatter/retrieval/registration suites PASS, 10 files / 272 tests.
- Full-suite first run: 4842 passed, one timing failure in untouched `tests/tools/event-loop-yields.test.ts`; isolated rerun passed. A subsequent full run under environment load had four unrelated integration timeouts, so it is recorded as degraded rather than attributed to this refactor.

## Review disposition

The adversarial pass returned six warnings about hardening opportunities already present in the monolith (output caps, recursive guards, table width validation/display width). They were moved verbatim and intentionally preserved to keep the refactor behavior-neutral; no fix-now finding or new regression was identified.
