## Findings (all from cross-provider adversarial — no primary findings)

### R-1 [RECOMMENDED] `derived-state-reducer` regex over-matches `async`/anything-containing-`sync`
- File: `src/tools/pattern-tools.ts:294`
- Confidence: 78/100  [CROSS:cursor-agent]
- Evidence: `regex: /…dispatch\s*\(\s*\{\s*type\s*:\s*['"][a-zA-Z_-]*sync/i`. The `i` flag + `[a-zA-Z_-]*sync` substring match makes `{type: 'asyncFoo'}` and `{type: 'asynchronous'}` (contains `sync` from index 1) match.
- Fix: anchor with word boundary or accepted action-name allowlist (e.g. `(?:^|_|-)sync(?:$|[^a-z])`); add a regression test asserting `{type: 'asyncSomething'}` does NOT match.

### R-2 [RECOMMENDED] `error-boundary-incomplete` description claims React "requires both" lifecycles — inaccurate
- File: `src/tools/pattern-tools.ts:355`
- Confidence: 72/100  [CROSS:cursor-agent]
- Evidence: Description: "React requires BOTH lifecycle methods for a complete ErrorBoundary". `componentDidCatch` + `setState` is a valid boundary; library boundaries deliberately split logging vs UI. Pattern itself flags valid code with a misleading reason.
- Fix: rewrite description as "recommended pairing for fallback UI + logging"; or scope regex to classes named `*ErrorBoundary*` only; add a `componentDidCatch` + `setState`-only test case that must NOT match.

### R-3 [RECOMMENDED] `rsc-non-serializable-prop-deep` — open-ended `[A-Z]\w*` constructor at `severity: critical`
- File: `src/tools/pattern-tools.ts:367`
- Confidence: 70/100  [CROSS:cursor-agent]
- Evidence: `new\s+(?:Map|Set|WeakMap|WeakSet|Symbol|RegExp|Promise|[A-Z]\w*)` — the trailing `[A-Z]\w*` matches any PascalCase constructor (`Error`, `URL`, plain DTO classes). Combined with `severity: critical`, this can flag legitimate RSC payloads as deploy-blocking.
- Fix: replace `[A-Z]\w*` with explicit denylist (`Date`, `Buffer`, `URL`, ...) and downgrade unknown PascalCase to `warning`; or split into `-deep` (denylist, critical) and `-deep-permissive` (catch-all, warning).

### R-4 [RECOMMENDED] severity-migration test uses hardcoded subset — Tier 6 + Tier 5 + non-React patterns escape
- File: `tests/tools/pattern-tools.test.ts:1747`
- Confidence: 80/100  [CROSS:cursor-agent]
- Evidence: The `Tier 6 — severity migration` test enumerates 28 specific React patterns. Future patterns added (Tier 7+) without `severity` would slip through; Tier 5 `derived-state` / `stale-closure-setstate` / `context-provider-value-inline` are NOT in the list either.
- Fix: derive list at runtime: `Object.entries(BUILTIN_PATTERNS).filter(([_, p]) => p.fileIncludePattern?.toString().includes("tsx"))` or equivalent tag, then assert `severity` defined for every match. Q-gate: prevents regressions on data-driven gate.

  - **R-6** `src/tools/pattern-tools.ts:328` `context-provider-value-via-variable` requires `;` between literal and JSX — misses ASI/multi-line forms; loosen terminator to `[;\n]` and add ASI test [CROSS:cursor-agent]
