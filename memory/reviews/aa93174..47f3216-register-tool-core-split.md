# Core tool registration split

The 35 core tool definitions retain their original public facade and exact
registration order while indexing, search, symbol, and metadata definitions
live in focused modules. Characterization, full-suite baseline, TypeScript
build, CodeSift checks, an independent CQ audit, and cross-provider review
covered the resulting production blobs.

arguments and rejects missing or blank symbol selectors. The final remediation
review found no issues.
