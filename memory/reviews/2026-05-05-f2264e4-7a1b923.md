## MUST-FIX

**R-1 [MUST-FIX]** Git hooks installer never reached from CLI  
**File:** `src/cli/commands.ts` (`handleSetup`), `src/cli/setup.ts` (`formatSetupLines` vs `setup`)  
**Confidence:** 98/100  
**Evidence:** `handleSetup` imports and calls only `formatSetupLines` (lines 454–478). `installGitHooks` exists solely inside `setup()` / `setupAll()` (`setup.ts` ~647–670). `formatSetupLines` (699–735) wires platform config, rules, and platform hook installers but **never** calls `installGitHooks`. End users running `codesift setup claude --hooks` therefore do not get the bundled global git post-commit behavior described in CHANGELOG.  
**Fix:** Either (a) change `handleSetup` to call `setup(platform, options)` / `setupAll(options)` and align output formatting, or (b) invoke `installGitHooks` from `formatSetupLines` when `options?.hooks && options.gitHooks !== false` (and mirror `setupAll`’s single tail call for `setup all`). Add an integration test that mocks `installGitHooks` and asserts it runs from the CLI path.

## RECOMMENDED

**R-2 [RECOMMENDED]** Overwrite of foreign global `core.hooksPath`  
**File:** `src/cli/git-hooks-installer.ts` ~156–172  
**Confidence:** 90/100 | **[CROSS:cursor-agent]**  
**Evidence:** If `git config --global --get core.hooksPath` returns a path ≠ `hooksDir`, `hooksPathConfigured` is false and the code runs `git config --global core.hooksPath` to the CodeSift dir — clobbering other tools’ global hook directories.  
**Fix:** If set and differs, skip (return `reason`) unless `force`, or prompt/document conflict.

**R-3 [RECOMMENDED]** Set `core.hooksPath` even when nothing was installed  
**File:** `src/cli/git-hooks-installer.ts`  
**Confidence:** 75/100 | **[CROSS:cursor-agent]**  
**Evidence:** Config update is unconditional after the install loop if path not already ours; if all scripts were skipped (e.g. missing bundle), Git may point at a partial directory.  
**Fix:** Only set global `core.hooksPath` when required files exist / `installed.length > 0` or equivalent.

**R-4 [RECOMMENDED]** `setup all` path never runs `setupAll`  
**File:** `src/cli/commands.ts` (~469–474)  
**Confidence:** 85/100  
**Evidence:** Platform `all` loops `formatSetupLines` per platform; does not call `setupAll`, so git-hook single-flight and any future `setupAll`-only logic stay bypassed. Same root cause as R-1.

**R-5 [RECOMMENDED]** Optional `chmod` repair when hash matches but `+x` lost

**Confidence:** 65/100 | **[CROSS:cursor-agent]**
